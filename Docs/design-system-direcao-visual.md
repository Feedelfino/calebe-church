# Documento 06 — Design System e Direção Visual

## Projeto

Calebe Church — Site Institucional Full-Stack

---

# 1. Objetivo do Documento

Este documento define a direção visual e os padrões de interface da aplicação Calebe Church.

O objetivo é manter consistência visual, organização da interface e padronização dos componentes utilizados no frontend do projeto.

---

# 2. Direção Visual do Projeto

A identidade visual da aplicação deverá transmitir:

- acolhimento;
- espiritualidade;
- modernidade;
- elegância;
- organização;
- autoridade visual.

A interface deverá equilibrar:

- visual institucional;
- experiência moderna;
- clareza visual;
- leitura confortável;
- responsividade.

---

# 3. Identidade Visual

Base visual principal:

- logo oficial da Calebe Church;
- paleta azul e dourado;
- estilo moderno e elegante;
- elementos visuais limpos;
- destaque para conteúdo institucional.

---

# 4. Paleta de Cores

## Cor Primária

Azul escuro institucional.

Exemplo previsto:

```txt
#0B1F3A
```

---

## Cor Secundária

Dourado institucional.

Exemplo previsto:

```txt
#D4AF37
```

---

## Cor de Fundo

Branco ou tons claros.

Exemplo previsto:

```txt
#FFFFFF
```

---

## Cor de Texto

Cinza escuro para leitura.

Exemplo previsto:

```txt
#1F2937
```

---

# 5. Tipografia

A tipografia deverá priorizar:

- leitura;
- elegância;
- modernidade;
- acessibilidade.

---

Alternativas:

---

# 6. Hierarquia Visual

A aplicação deverá possuir hierarquia clara entre:

- títulos;
- subtítulos;
- textos;
- botões;
- destaques.

---

## Títulos Principais

Maior destaque visual.

Exemplo:

```txt
Hero Title
Nome da Igreja
Chamadas principais
```

---

## Subtítulos

Utilizados para separar seções.

Exemplo:

```txt
Ministérios
Cultos
NAF
```

---

## Texto Institucional

Utilizado para descrições e conteúdos.

Características previstas:

- leitura confortável;
- contraste adequado;
- espaçamento equilibrado.

---

# 7. Estrutura Visual da Interface

A interface deverá seguir:

- alinhamento consistente;
- espaçamentos organizados;
- separação clara de seções;
- visual limpo;
- navegação intuitiva.

---

# 8. Estratégia de Componentização Visual

Os componentes deverão manter:

- consistência visual;
- reutilização;
- padronização de espaçamentos;
- padronização de cores;
- padronização tipográfica.

---

# 9. Botões

Os botões deverão possuir:

- contraste adequado;
- destaque visual;
- acessibilidade;
- responsividade;
- estados visuais.

---

## Estados previstos

```txt
Default
Hover
Active
Disabled
```

---

# 10. Cards

Os cards deverão ser utilizados para:

- ministérios;
- cultos;
- cursos;
- informações institucionais.

Características previstas:

- bordas suaves;
- espaçamento interno;
- sombra leve;
- boa leitura.

---

# 11. Navbar

A navbar deverá possuir:

- logo da igreja;
- navegação clara;
- menu responsivo;
- acesso rápido às redes sociais.

Características previstas:

- fixa no topo;
- responsiva;
- visual limpo.

---

# 12. Footer

O footer deverá possuir:

- identidade institucional;
- endereço;
- horários;
- redes sociais;
- direitos reservados.

---

# 13. Responsividade

A interface deverá funcionar corretamente em:

- celulares;
- tablets;
- notebooks;
- desktops.

---

# 14. Estratégia Mobile First

O desenvolvimento visual deverá iniciar pelo mobile.

Objetivos:

- melhorar experiência mobile;
- melhorar SEO;
- garantir acessibilidade;
- facilitar responsividade.

---

# 15. Estratégia de UX

A experiência do usuário deverá priorizar:

- clareza;
- simplicidade;
- leitura fácil;
- navegação intuitiva;
- carregamento rápido;
- destaque para informações importantes.

---

# 16. Estratégia de UI

A interface deverá manter:

- identidade consistente;
- organização visual;
- contraste adequado;
- harmonia entre cores;
- equilíbrio entre texto e imagens.

---

# 17. Estratégia de Acessibilidade
