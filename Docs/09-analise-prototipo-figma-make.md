# Documento 09 — Análise do Protótipo Figma Make

## Projeto

Calebe Church — Site Institucional Full-Stack

---

# 1. Objetivo do Documento

Este documento registra a análise inicial do protótipo criado no Figma Make para o projeto Calebe Church.

O objetivo é definir o papel desse material dentro do projeto, identificar o que será aproveitado como referência e deixar claro que o desenvolvimento oficial será feito manualmente no VS Code.

---

# 2. Contexto

Foi gerado um protótipo utilizando o Figma Make com base no briefing institucional da Calebe Church.

O material criado apresenta uma estrutura visual e funcional próxima de uma aplicação web, utilizando seções compatíveis com o projeto planejado.

Entretanto, o arquivo gerado não substitui a documentação oficial nem o desenvolvimento próprio do projeto.

---

# 3. Papel do Protótipo Figma Make

O protótipo será utilizado como:

- referência visual;
- referência estrutural;
- apoio para comparação de layout;
- apoio para organização das seções;
- inspiração para componentes;
- material auxiliar de UX/UI.

O protótipo não será utilizado como base oficial de código.

---

# 4. Motivo para Não Utilizar como Base Oficial

O objetivo principal deste projeto é desenvolver a aplicação do zero, com foco em aprendizado prático, portfólio e domínio técnico.

Por isso, o desenvolvimento oficial será feito manualmente utilizando:

- VS Code;
- React;
- TypeScript;
- Python;
- FastAPI;
- SQL;
- Git;
- GitHub.

Essa decisão permite maior aprendizado sobre:

- estrutura de projeto;
- componentização;
- lógica de programação;
- consumo de API;
- backend;
- banco de dados;
- segurança;
- QA;
- deploy.

---

# 5. Componentes Identificados no Protótipo

O protótipo gerado possui seções compatíveis com a estrutura planejada para o projeto.

Componentes/seções identificados:

```txt
Navbar
HeroSection
AboutSection
ServicesSection
MinistriesSection
NAFSection
SocialMediaSection
LocationSection
Footer
```

Esses componentes poderão servir como referência para a criação dos componentes React no projeto oficial.

Relação prevista entre o protótipo e o frontend oficial:


---

# 7. O Que Será Aproveitado

Serão aproveitados como referência:

- ordem das seções;
- ideia geral de layout;
- distribuição visual;
- estrutura de componentes;
- hierarquia das informações;
- possíveis soluções visuais;
- experiência inicial do usuário.

---

# 8. O Que Não Será Aproveitado Diretamente

Não será aproveitado diretamente:

- código gerado automaticamente;
- estrutura automática do Figma Make;
- componentes sem revisão;
- estilos sem padronização;
- decisões visuais que não estejam alinhadas aos documentos oficiais.

---

# 9. Fonte Oficial do Projeto

A fonte oficial de decisão do projeto será composta por:

```txt
Documentação oficial
Protótipo revisado
Código desenvolvido no VS Code
Repositório GitHub
```

O Figma Make será considerado apenas uma referência auxiliar.

---

# 10. Critérios para Validação Visual

Antes de iniciar o frontend, o protótipo deverá ser avaliado considerando:

- clareza da navegação;
- hierarquia visual;
- responsividade;
- contraste;
- legibilidade;
- alinhamento com identidade visual;
- coerência com o briefing;
- facilidade de transformação em componentes React.

---

# 11. Decisão Técnica

A decisão oficial é reconstruir o projeto manualmente, utilizando o protótipo Figma Make apenas como referência.

O desenvolvimento será feito do zero para garantir aprendizado técnico, controle da arquitetura e qualidade do código.
